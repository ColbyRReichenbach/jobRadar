# Archive

This folder holds documents that were useful while AppTrail was being designed, audited, or shipped, but no longer belong in the active documentation set.

## What Lives Here

- `planning/`: product specs, sprint plans, gap lists, brainstorm notes, and old deployment plans
- `audits/`: one-off review documents and implementation audits
- `extension/`: retired extension copy that has been replaced by the current store materials
- `tooling/`: old tool-specific working instructions that should not drive the current repo

## Why They Were Moved

These files still have historical value, but they were cluttering the top-level repo and making it harder to tell which documents describe the product as it exists today.

The active documentation set now lives in:

- [README.md](../../README.md)
- [docs/README.md](../README.md)
- [TECHNICAL.md](../../TECHNICAL.md)
- [SECURITY.md](../../SECURITY.md)
- [deployment-checklist.md](../deployment-checklist.md)
- [privacy-policy.md](../privacy-policy.md)

## Archive Rule

Move a document here when it is still useful as history but should not be read as the current source of truth. Examples include completed branch plans, superseded product specs, old audits, one-off execution checklists, and tool-specific notes from earlier build phases.
